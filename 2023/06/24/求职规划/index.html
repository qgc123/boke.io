2.简历指导

简历字数并非越多越好，任何关键词都可能引起面试官的考察兴趣

  • 基础信息：

    • 书写顺序：个人信息->教育经历（获奖经历）->专业技能->实习（而非工作）经历->项目经验
    • 文件名：姓名-院校-电话
    • 文件格式：pdf（word排版会乱）
    • 模版：冷色调、不要表格

  • 基本信息：求职意向、电话、出生年月、邮箱、照片

    • 个人博客/Github视情况而定
    • 邮箱：必须为常用邮箱
    • 意向城市：强需求可以填写

  • 教育经历：学校、专业、学历、时间区间

    • 主修课程：经得住问，可以写
    • 竞赛经历：算法题难度增加

  • 专业技能：精通 > 熟悉 > 熟练使用=掌握 > 了解

    • 专注
      • 不能同时熟悉Java、C++
    • 不能同时熟悉MySQL、Oracle
    • 不要存在单词错误、大小写错误
    • 结构层次、分门别类
    • 数据结构与算法不建议提及
    • 不建议出现的关键词：深入、精通、扎实、经验丰富

  • 自我评价：不需要

目的：让面试官快速了解项目背景，与面试官有的聊

  • 项目来源：【1-2要标明清楚】

    • 实习经历的项目

    • 学校的生产项目（学校的XX系统）

    • 学校的课程设计 或者 练手项目

  • 项目数量：2-3个

  • 项目 = 产品 + 技术 + 结果（传达给简历阅读者的信息）

    • 项目描述：做了什么产品
    • 技术栈：用到了什么技术（与专业技能相辅相成）
    • 取得了什么结果
      • 规模（量化指标）：用户规模，资金规模等
        • 若规模较大/场景较为明显，要想好“如何支撑”
      • 角色：主导者、参与者
        • 作为参与者，可以适当包装，但要清楚细节
        • 作为主导者，可以讲明带领的团队人数
      • 推荐描述方式：
        • 通过XXX实现XXX，达到XX效果

  • 项目书写其他技巧：

    • 按照时间倒序写
    • 详细的项目描述将亮点写在前面
    • 岗位无关的项目尽量少写

  • 建议的项目描述模版

项目名称 所属组织/公司 时间区间

项目描述： 2-5句话内描述清楚主要功能。

角色：参与者/主导者

技术栈：A、B、C、D、E、F（重要的放前面）

  1. 通过X实现了Y，达到了Z的效果

  2. 通过X实现了Y，达到了Z的效果

  3. 通过X实现了Y，达到了Z的效果

  4. 通过X实现了Y，达到了Z的效果

[个数：3-6]

书籍推荐

关于看书学习/上网学习的几点建议：

  1. 写的比较好的书是前后逻辑是很强的，利于理解。网上的博文的作者不一定有毅力坚持，往往烂尾，且天下文章一大抄，其中的错误可能被广为流传。
  2. 背面试题 vs 技术理解：一个是招式，一个是内功，相辅相成，缺一不可。
  3. 购买前查看简介，基于的版本，是不是如今的主流版本。
  4. 针对于“比较新的又没有大规模实践的”技术的书籍可以不买，看官方文档。

刷题：

  • 理解：《漫画算法 小灰的算法之旅》
  • 刷题：《剑指Offer》

Java基础：

  • 大杂烩基础：《Effective Java》（第三版）
  • 并发：《Java并发编程实战》
  • JVM：《深入理解Java虚拟机》（第三版）

Mysql：

  • 进阶：《MySQL技术内幕:InnoDB存储引擎》

Linux：

  • 鸟哥的Linux私房菜（基础学习篇）

视野拓宽与技术理解：

  • 《码农翻身》
  • 《代码整洁之道》
  • 《重构：改善既有代码的设计》
  • 《领域驱动设计》
